# Hermetic Fake-Fleet for Local Router Testing ‚Äî Design

- **Date:** 2026-06-16
- **Status:** Approved (brainstorm) ‚Äî revised after code-grounded review ‚Äî pending implementation plan
- **Scope:** `genefoundry-router` only
- **Related:** `docs/specs/2026-06-13-genefoundry-router-design.md`

## 1. Problem

The router federates ~12 active `*-link` MCP backends. Validating it locally currently
means either (a) the in-process fixtures in `tests/integration/conftest.py` (which never
exercise the HTTP wire), or (b) standing up the real fleet as Docker containers ‚Äî heavy
(pgvector for pubtator/genereviews, data volumes, ~15 services) and not per-change material.

We want a **third tier**: run the *real* router locally, federating impersonated backends
over **real Streamable-HTTP**, fully **offline**, deterministic, with **no Docker**.

## 2. Goals / Non-goals

**Goals**
- Run the unmodified router against a faked fleet over real Streamable-HTTP on localhost.
- 100% offline and deterministic at test/dev time (no network, no containers).
- High fidelity: fakes expose the real tool **names, descriptions, and inputSchema** so that
  namespacing, collision handling, and **BM25 search** behave as in production (see ¬ß3.1).
- One process, not 15. `make dev-fleet` to start; `make test-e2e` one-shot.
- A refreshable mechanism to keep fidelity current as the fleet evolves.

**Non-goals (deferred ‚Äî ¬ß11)**
- Live-VPS smoke (`make smoke-remote`); faithful local Docker repro.
- Argument-remap `transform` blocks ‚Äî the fleet is Tool-Naming Standard v1 clean, so none
  exist. The retained `strip_prefix` machinery stays covered by a synthetic non-compliant
  fixture at the integration layer (`tests/integration/test_strip_prefix.py`), not the fleet.

## 3. Approach: committed snapshot (record / replay)

Fakes are driven by a **committed manifest** of the fleet's tool surface. Because it is read
from disk, every test/dev run is offline regardless of how the manifest was produced. The
manifest is regenerated by a deliberate, occasional **online** refresh tool ‚Äî the golden-file
/ VCR-cassette pattern. Chosen over hand-authored stubs: equally offline at run time, higher
fidelity, no hand-drift across 12 backends. The fleet just standardized on Tool-Naming
Standard v1, so a snapshot captures the canonical surface and doubles as documentation and a
regression oracle.

**Fidelity caveat (sequencing):** a snapshot reflects what is *deployed*; merged-to-GitHub ‚âÝ
deployed-to-VPS. Refresh after `strato` redeploys v1, or run against backends locally. The
manifest records provenance (`source`, `captured_at`, per-backend `version`) so staleness is
visible.

### 3.1 Validated facts (installed fastmcp 3.4.2 ‚Äî verified, not assumed)

These drive several design decisions; confirmed against the installed package and repo:

- **BM25 search text = name + description + parameter-names + parameter-descriptions. NOT
  tags, NOT outputSchema/annotations** (`fastmcp/server/transforms/search/base.py:
  _extract_searchable_text`). ‚áí fakes must reproduce the real `inputSchema`, and search
  assertions target description/param text, not tags.
- **`build_app` always applies the search transform**, which *replaces* the full tool listing
  with `search_tools` + `call_tool` + pinned essentials (`server.py:88`, `tool_search.py`).
  Pinned defaults: `gnomad_resolve_variant_id`, `gnomad_search_genes`. ‚áí full-catalog
  assertions must use `build_server(enable_search=False)` (the `cli.py:_list_federated_tools`
  pattern); the searched build is tested separately.
- **Mounted FastMCP `http_app()` instances require their `lifespan` entered** or the
  session manager never initializes (mirrors `server.py:79,86`). ‚áí the fleet's outer
  Starlette app must enter *every* child `mcp_app.lifespan` via `AsyncExitStack`.
- **Remote clients expose FastMCP tags under `tool.meta["fastmcp"]["tags"]`**, not
  `tool.tags` (confirmed by `tests/integration/test_strip_prefix.py:29`). ‚áí tag-presence
  assertions read `meta`.
- **The CLI `run` has no `--env-file`; backend URLs resolve from `os.environ`** via
  `load_registry(servers_file, os.environ)` (`cli.py:61`). ‚áí `make run-dev` exports the dev
  env into the process; it does not pass a flag.
- **`.gitignore` ignores `.env.*`** (line 13). ‚áí committing `.env.dev` needs an explicit
  `!.env.dev` negation.

## 4. Components

1. **`genefoundry_router/devtools/fake_fleet.py`** (runnable module, < 600 LOC)
   - Reads the manifest; builds one echo-`FastMCP` per backend with the real
     names/descriptions/**reconstructed inputSchema** (¬ß6.1); mounts them onto **one Starlette
     app, one localhost port**, path-routed at `/<namespace>/mcp`.
   - **Composed lifespan:** an outer lifespan enters every child `mcp_app.lifespan` via
     `contextlib.AsyncExitStack` (required ‚Äî ¬ß3.1).
   - Entry: `python -m genefoundry_router.devtools.fake_fleet [--port 9100] [--manifest PATH]`.
   - Importable so the e2e fixture builds and serves the app in-process.

2. **`genefoundry_router/devtools/fakes.py`** (shared builder)
   - `make_fake_backend(...)` promoted out of `conftest.py`; `conftest.py` re-exports it so the
     harness and existing integration tests share one builder.

3. **`tests/fixtures/fleet_manifest.json`** ‚Äî committed snapshot; the only run-time data.

4. **`scripts/snapshot_fleet.py`** (on-demand online refresh, < 600 LOC; never run in tests)
   - Connects to each backend (URLs from an env file, default `.env`), lists tools
     (name + description + inputSchema + outputSchema/annotations if present), and records
     `version` from the **MCP `initialize` handshake `serverInfo.version`** (portable contract;
     not an optional `get_server_capabilities` tool). Captures FastMCP tags from
     `tool.meta["fastmcp"]["tags"]`. Per-backend resilient: an unreachable backend keeps its
     prior entry with a warning ‚Äî never clobbered with an empty toolset.

5. **`servers.dev.yaml` + `.env.dev`** (committed dev config; `.env.dev` un-ignored via
   `!.env.dev` ‚Äî safe: localhost URLs only, `GF_AUTH_MODE=none`, no secrets)
   - Mirror `servers.yaml` but with every `GF_*_URL` ‚Üí `http://127.0.0.1:9100/<namespace>/mcp`.
     The fleet script derives the same `<namespace>` paths from `servers.yaml`, giving one
     port/path contract. The script validates `.env.dev`/`servers.dev.yaml` against its derived
     paths: **warn** under `dev-fleet`, **hard-fail** under `test-e2e` (¬ß9). It never rewrites
     env files.

6. **Make targets**
   - `make dev-fleet` ‚Äî run the fakes; print the `<namespace> ‚Üí URL` table.
   - `make run-dev` ‚Äî `set -a; . ./.env.dev; set +a; uv run genefoundry-router run
     --servers-file servers.dev.yaml` (env exported into `os.environ`; no `--env-file`).
   - `make test-e2e` ‚Äî pytest `tests/e2e/`: boot fleet in-process ‚Üí real-wire assertions ‚Üí
     teardown.
   - `make snapshot-fleet` ‚Äî deliberate online refresh (`scripts/snapshot_fleet.py`); rewrites
     the manifest. Not part of any CI loop.

## 6. Manifest schema

```json
{
  "snapshot_meta": {
    "captured_at": "2026-06-16T09:00:00Z",
    "source": "live",                      // "live" | "local"
    "router_servers_file": "servers.yaml"
  },
  "backends": {
    "gnomad": {
      "version": "5.0.0",                  // from initialize.serverInfo.version
      "tools": [
        {
          "name": "resolve_variant_id",    // leaf name (post-v1, unprefixed)
          "description": "Resolve ‚Ä¶",
          "inputSchema": { "type": "object",
            "properties": { "variant_id": { "type": "string", "description": "rsID or ‚Ä¶" } },
            "required": ["variant_id"] },
          "outputSchema": null,            // captured if present
          "annotations": null,             // captured if present
          "tags": ["variant", "gene"]      // surfaced, NOT searched
        }
      ]
    }
  }
}
```

- `inputSchema` is **required** for fidelity: BM25 indexes parameter names + descriptions
  (¬ß3.1). `tags` are captured for surfacing/assertions but are not part of search text.
- The manifest MUST contain the pinned essentials' leaves (`gnomad/resolve_variant_id`,
  `gnomad/search_genes`) or the pinned-visibility assertion (¬ß8) cannot pass.

### 6.1 Fake tool reconstruction

Each fake tool must reproduce its captured `inputSchema` so the gateway's search index matches
production. Two mechanisms, in preference order (the exact one is pinned during planning by an
import smoke against the installed fastmcp, per AGENTS.md):

1. **Direct schema injection** ‚Äî construct the FastMCP `Tool` with the captured JSON Schema if
   the installed API supports an explicit `parameters`/schema argument (exact reproduction).
2. **Synthesized signature** ‚Äî build a function whose `__signature__`/annotations carry the
   captured params as `Annotated[type, Field(description=...)]`, then `Tool.from_function`
   (FastMCP derives the same schema). Echo body returns `{tool, server, args}`.

## 7. Run ergonomics

```bash
make dev-fleet      # terminal 1: fakes on :9100, prints namespace‚ÜíURL map
make run-dev        # terminal 2: exports .env.dev, runs router on :8000 vs the fakes
# ‚Üí point Claude/Gemini/any MCP client at http://127.0.0.1:8000/mcp

make test-e2e       # one-shot: boots fleet in-process, asserts, tears down
make snapshot-fleet # deliberate refresh (online) ‚Üí rewrites fleet_manifest.json
```

## 8. Testing strategy (`tests/e2e/`)

A session-scoped fixture serves the fake-fleet Starlette app via an in-process `uvicorn.Server`
on an ephemeral port (readiness-polled), with the fleet's composed lifespan entered, then shuts
down. Two complementary checks (both offline, deterministic):

**(a) Full catalog ‚Äî search disabled.** `build_server(settings, dev_registry, enable_search=False)`
with backend URLs pointed at the fleet; connect an in-process `Client(server)` (the gateway‚Üífake
hop is real HTTP). Assert: every backend leaf surfaces as `<namespace>_<leaf>`; no collisions;
the federated set equals the manifest projection ‚Äî **the no-transform regression guard** (no
double-prefix, names match v1 exactly).

**(b) Search + serving ‚Äî production build.** `build_app` served over uvicorn; connect a `Client`
over **HTTP** (client‚Üígateway‚Üífake all on the wire). Assert: `search_tools` returns expected
hits by **description / parameter text** (not tags); pinned essentials are present in
`list_tools()`; `call_tool` (or a direct namespaced call) round-trips end-to-end. Tag presence,
where asserted, is read from `tool.meta["fastmcp"]["tags"]`.

`make test-e2e` is offline and deterministic, so it MAY be wired into a `make ci-full`; it is
kept out of the fast `make ci-local` to preserve that loop's speed (¬ß10).

## 9. Error handling

| Condition | Behaviour |
|---|---|
| Manifest missing / malformed | Fleet exits non-zero, pointing at `make snapshot-fleet`. |
| `--port` already in use | Clear error; port overridable. |
| `servers.dev.yaml`/`.env.dev` ‚Üî derived-path mismatch | **dev-fleet:** warn + continue. **test-e2e:** hard fail. Never auto-edit env files. |
| Backend disabled in `servers.yaml` (e.g. hgnc) | Fleet serves whatever is in the manifest; the router (`servers.dev.yaml` `enabled`) is authoritative on what it mounts. |
| Snapshot of an unreachable backend | Keep prior manifest entry + warn; never write an empty toolset. |
| Pinned essential missing from manifest | `test-e2e` fails fast with a clear message (its leaf must exist). |

## 10. Decisions made

- **Committed snapshot over hand-authored stubs** (¬ß3).
- **One Starlette app, path-routed, single port, child lifespans entered via AsyncExitStack**
  over N ports / Docker.
- **Fakes reproduce `inputSchema`** (param text is the searchable surface); tags captured but
  not searchable (¬ß3.1).
- **Two-layer e2e:** `build_server(enable_search=False)` for full-catalog/collision/no-transform;
  `build_app` over HTTP for search + serving + invocation.
- **`make run-dev` exports `.env.dev`** into the env (no CLI `--env-file`); `.env.dev` committed
  via `!.env.dev` (no secrets).
- **Version provenance = `initialize.serverInfo.version`**, not a backend tool.
- **`make test-e2e` stays out of `make ci-local`** (offline but adds boot time); optionally
  under `make ci-full`. Config mismatch is a **hard fail** in e2e.

## 11. Future extensions (out of scope here)

A `make smoke-remote` reusing `servers.yaml` + real `.env` (live-fleet smoke), and a profiled
`docker/docker-compose.fleet.yml` for faithful single-backend debugging. Both are additive and
share the manifest/assertions where useful.

## 12. Boundary

Research use only; the fakes echo synthetic payloads and carry no real data. The harness
mirrors the backends' disclaimers and adds no clinical capability.
