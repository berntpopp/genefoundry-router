# Hermetic Fake-Fleet for Local Router Testing ‚Äî Design

- **Date:** 2026-06-16
- **Status:** Approved (brainstorm) ‚Äî pending implementation plan
- **Scope:** `genefoundry-router` only
- **Related:** `docs/specs/2026-06-13-genefoundry-router-design.md`

## 1. Problem

The router federates ~12 active `*-link` MCP backends. Validating it locally currently
means either (a) the in-process fixtures in `tests/integration/conftest.py` (which never
exercise the HTTP wire), or (b) standing up the real fleet as Docker containers ‚Äî which is
heavy (pgvector for pubtator/genereviews, data volumes, ~15 services) and not something to
run on every change.

We want a **third tier**: run the *real* router locally, federating impersonated backends
over **real Streamable-HTTP**, fully **offline**, deterministic, and with **no Docker**.

## 2. Goals / Non-goals

**Goals**
- Run the unmodified router against a faked fleet over real Streamable-HTTP on localhost.
- 100% offline and deterministic at test/dev time (no network, no containers).
- High fidelity: fakes expose the **real tool names, descriptions, and tags** so namespacing,
  collision handling, and BM25 search behave as they do in production.
- One process, not 15. Single command to start (`make dev-fleet`) and a one-shot e2e
  (`make test-e2e`).
- A refreshable mechanism to keep fidelity current as the fleet evolves.

**Non-goals (explicitly deferred ‚Äî see ¬ß11)**
- Smoke-testing against the live VPS fleet (`make smoke-remote`).
- Faithful local Docker reproduction of a backend (pgvector, real data).
- Testing argument-remap `transform` blocks ‚Äî the fleet is Tool-Naming Standard v1 clean,
  so no router-side transforms exist. The retained `strip_prefix` machinery in
  `normalization.py` stays covered by a synthetic non-compliant fixture at the integration
  layer (`tests/integration/test_strip_prefix.py`), **not** by the snapshot fleet.

## 3. Approach: committed snapshot (record / replay)

The fakes are driven by a **committed manifest** of the fleet's tool surface. Because the
manifest is read from disk, every test and dev run is offline regardless of how the manifest
was produced. The manifest is generated by a deliberate, occasional **online** refresh tool ‚Äî
the golden-file / VCR-cassette pattern.

This was chosen over hand-authored stubs: both are equally offline at run time, but the
snapshot is higher fidelity and does not drift by hand across 12 backends. Timing is
favourable ‚Äî the fleet just standardized on Tool-Naming Standard v1, so a snapshot captures
the canonical surface and doubles as living documentation and a regression oracle.

### Fidelity caveat (sequencing)
A snapshot reflects what is **deployed**, and merged-to-GitHub ‚âÝ deployed-to-VPS. The refresh
step must run either *after* `strato` redeploys the v1 versions, or against the backend repos
run locally. The committed manifest records its provenance (`source`, `captured_at`,
per-backend `version`) so staleness is visible.

## 4. Components

1. **`genefoundry_router/devtools/fake_fleet.py`** (runnable module, < 600 LOC)
   - Reads the manifest, builds one echo-`FastMCP` per backend (real names/descriptions/tags,
     trivial echo bodies), mounts them all onto **one Starlette app on one localhost port**,
     path-routed at `/<namespace>/mcp`.
   - Entry: `python -m genefoundry_router.devtools.fake_fleet [--port 9100] [--manifest PATH]`.
   - Importable: the e2e fixture constructs the app and serves it in-process.

2. **`genefoundry_router/devtools/fakes.py`** (shared builder)
   - `make_fake_backend(name, tools)` promoted out of `conftest.py` so the harness *and* the
     existing integration tests use one builder (`conftest.py` re-exports from here).

3. **`tests/fixtures/fleet_manifest.json`** (committed snapshot ‚Äî offline source of truth)
   - Schema in ¬ß6. The only data the harness needs at run time.

4. **`scripts/snapshot_fleet.py`** (on-demand online refresh, < 600 LOC; never run in tests)
   - Connects to each backend (URLs from an env file, default `.env`), lists tools
     (name + description + tags + backend version via `get_server_capabilities` where present),
     writes the manifest. Per-backend resilient: an unreachable backend keeps its prior entry
     and emits a warning ‚Äî never clobbered with an empty toolset.

5. **`servers.dev.yaml` + `.env.dev`** (committed dev config)
   - Mirror `servers.yaml` but with every `GF_*_URL` pointing at
     `http://127.0.0.1:9100/<namespace>/mcp`. The fleet script derives the same `<namespace>`
     paths from `servers.yaml`, giving a single port/path contract. The script validates that
     `.env.dev`/`servers.dev.yaml` agree with its derived paths and **warns on mismatch**; it
     does **not** auto-mutate env files (avoids clobbering local edits).

6. **Make targets**
   - `make dev-fleet` ‚Äî run the fakes; prints the served `<namespace> ‚Üí URL` table.
   - `make run-dev` ‚Äî run the router against the fakes (`GF_SERVERS_FILE=servers.dev.yaml`,
     `--env-file .env.dev`).
   - `make test-e2e` ‚Äî pytest `tests/e2e/`: boot fleet in-process ‚Üí real-client assertions ‚Üí
     teardown.
   - `make snapshot-fleet` ‚Äî deliberate online refresh (`scripts/snapshot_fleet.py`); rewrites
     `tests/fixtures/fleet_manifest.json`. Not part of any CI loop.

## 6. Manifest schema

```json
{
  "snapshot_meta": {
    "captured_at": "2026-06-16T09:00:00Z",
    "source": "live",                      // "live" | "local"
    "router_servers_file": "servers.yaml"
  },
  "backends": {
    "gnomad": {
      "version": "5.0.0",
      "tools": [
        { "name": "get_variant_details", "description": "‚Ä¶", "tags": ["variant", "gene"] },
        { "name": "search_genes",         "description": "‚Ä¶", "tags": ["gene"] }
      ]
    }
  }
}
```

- `name` is the **leaf** name as the backend exposes it (post-v1, unprefixed). The router
  applies namespacing; the fake does not pre-namespace.
- `description` + `tags` are captured so BM25 `search_tools` assertions are meaningful
  (the search transform indexes name + description + tags).

## 7. Run ergonomics

```bash
make dev-fleet      # terminal 1: fakes on :9100, prints namespace‚ÜíURL map
make run-dev        # terminal 2: router on :8000 against the fakes
# ‚Üí point Claude/Gemini/any MCP client at http://127.0.0.1:8000/mcp

make test-e2e       # one-shot: boots fleet in-process, asserts, tears down
make snapshot-fleet # deliberate refresh (online) ‚Üí rewrites fleet_manifest.json
```

## 8. Testing strategy (`tests/e2e/`)

A session-scoped fixture serves the fake-fleet Starlette app via an in-process `uvicorn.Server`
on an ephemeral port (readiness-polled on one mount's `/health`), yields the base URL, then
shuts down. The router is built against `servers.dev.yaml` rewritten to that base URL.

Assertions over the real wire:
- Every backend's leaf tool surfaces as `<namespace>_<leaf>` (namespacing).
- No tool-name collisions across the federated surface.
- `search_tools` returns expected hits by tag/description; pinned essentials always visible.
- `/health` is `healthy`.
- **Regression guard:** no backend needs a `transform` block ‚Äî i.e. no double-prefixing and
  the federated names match the v1 manifest exactly.

`make test-e2e` is offline and deterministic, so it MAY also be wired into a `make ci-full`
target; it is kept out of the fast `make ci-local` to preserve that loop's speed (decision ¬ß10).

## 9. Error handling

| Condition | Behaviour |
|---|---|
| Manifest missing / malformed | Fleet exits non-zero with a message pointing at `make snapshot-fleet`. |
| `--port` already in use | Clear error; port is overridable. |
| `servers.dev.yaml`/`.env.dev` ‚Üî derived paths mismatch | Warn (loud), continue; never auto-edit env files. |
| Backend disabled in `servers.yaml` (e.g. hgnc) | Fleet serves whatever is in the manifest; the router (via `servers.dev.yaml` `enabled`) is authoritative on what it mounts. |
| Snapshot of an unreachable backend | Keep prior manifest entry, warn; do not write an empty toolset. |

## 10. Decisions made

- **Committed snapshot over hand-authored stubs** (¬ß3).
- **One Starlette app, path-routed, single port** over N ports or Docker ‚Äî simplest offline
  topology with a real wire.
- **`.env.dev`/`servers.dev.yaml` are committed and authoritative; the fleet script validates
  but never rewrites them.**
- **`make test-e2e` stays separate from `make ci-local`** (offline but adds boot time);
  optionally aggregated under `make ci-full`.

## 11. Future extensions (out of scope here)

The deferred tiers slot in cleanly later: a `make smoke-remote` reusing `servers.yaml` +
real `.env` (live-fleet smoke), and a profiled `docker/docker-compose.fleet.yml` for faithful
single-backend debugging. Both are additive and share the manifest/assertions where useful.

## 12. Boundary

Research use only; the fakes echo synthetic payloads and carry no real data. The harness
mirrors the backends' disclaimers and adds no clinical capability.
