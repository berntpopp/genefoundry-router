# AutoPVS1 #76: preserve the selected clinical-validity option

> Historical record — this document records the design or plan as of its date. Current behavior is
> defined by implemented code, standards, release evidence, and tests.

> **For the implementer:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task by task.

**Goal:** Ensure AutoPVS1 disease-mechanism results expose only the selected clinical-validity option, never the concatenation of every option in an HTML `<select>`.

**Architecture:** Keep parsing at the HTML boundary in `autopvs1_link/api/autopvs1_parsers.py`. Normalise either the selected option or a static table cell into one stable string before constructing `DiseaseMechanism`; represent a malformed/unselected select as the documented `"not_available"` sentinel. Preserve the existing model and MCP response envelope so the public `get_variant_pvs1_data` tool simply serialises the corrected parser result.

**Tech Stack:** Python 3.12, Beautiful Soup, Pydantic, FastMCP 3.x, pytest/pytest-asyncio, uv, Docker.

## Task 2: Extract selected option text at the HTML boundary

**Files:**

- Modify: `autopvs1_link/api/autopvs1_parsers.py`
- Modify: `tests/unit/test_scraper_parsers.py`

**Step 1: Implement the smallest parser helper.**

Near the existing parser text-normalisation helpers in `autopvs1_link/api/autopvs1_parsers.py`, define the module constant:

~~~python
CLINICAL_VALIDITY_UNAVAILABLE = "not_available"
~~~

Add a private helper that accepts the fourth Beautiful Soup table-cell tag and behaves as follows:

1. If it has no descendant `<select>`, return the collapsed, stripped cell text, or `CLINICAL_VALIDITY_UNAVAILABLE` if blank.
2. If it has a `<select>`, inspect only that select's direct/descendant `<option>` elements and select the option for which `option.has_attr("selected")` is true.
3. Return collapsed, stripped selected-option text if it is non-empty.
4. If no option is selected, or selected text is empty, return `CLINICAL_VALIDITY_UNAVAILABLE`.

Do not fall back to the first option and do not call `.text` on the select. AutoPVS1's markup denotes the browser-selected value with `selected`; inventing a choice would incorrectly state clinical evidence.

**Step 2: Wire the helper into the model construction.**

In `parse_disease_mechanisms`, replace only:

~~~python
clinical_validity=cols[3].text.strip(),
~~~

with a call to the new helper. Do not alter field names, the Pydantic `DiseaseMechanism` model, rate limiting, request code, or unrelated parser rows.

**Step 3: Re-run focused parser tests and confirm GREEN.**

Run:

~~~bash
uv run pytest tests/unit/test_scraper_parsers.py -q
~~~

Expected result: all parser tests pass; the selected middle option is exactly `"No Reported Evidence"`, static text is exactly `"Strong"`, and an invalid no-selection cell is exactly `"not_available"`.

**Step 4: Refactor only if it improves local clarity.**

If the helper duplicates a pre-existing text collapse helper, reuse it. Do not generalise the parser beyond select cells, and retain type annotations compatible with the installed Beautiful Soup types and the repository's mypy configuration.

**Step 5: Commit the parser repair.**

~~~bash
git add autopvs1_link/api/autopvs1_parsers.py tests/unit/test_scraper_parsers.py
git commit -m "fix: preserve selected AutoPVS1 clinical validity"
~~~
